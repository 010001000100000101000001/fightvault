# FightVault

FightVault is your ultimate destination for the latest news, updates, and information on MMA and combat sports. The platform aims to keep fans and enthusiasts informed and connected with the combat sports community. Whether you are a fighter, coach, or fan, FightVault provides valuable content to keep you engaged with the world of MMA and combat sports.

## Project Overview

FightVault is a comprehensive platform designed for MMA and combat sports enthusiasts. It aims to bring together a community of passionate individuals who share a love for all things martial arts. The platform is dedicated to providing a space where fans can write, read, and learn about their favorite sports and fighters. FightVault strives to offer a comprehensive and engaging experience, from in-depth articles and interviews to the latest news and event coverage.

## User Stories

### View a Page of Posts

**As a Site User, I can view a list of recent posts on the home page so that I can quickly find new content.**

- **Acceptance criteria 1:** The home page displays a list of recent posts, including the title, author, date, and a brief summary.
- **Acceptance criteria 2:** Users can click on the title or summary of each post to read the full post.
- **Acceptance criteria 3:** Pagination is implemented to navigate through multiple pages of posts.

### Open a Post

**As a Site User, I can click on a post so that I can read the full text.**

- **Acceptance criteria 1:** When a blog post title is clicked, a detailed view of the post is seen.

### View Comments

**As a Site User / Admin, I can view comments on an individual post so that I can read the conversation.**

- **Acceptance criteria 1:** Given one or more user comments, the admin can view them.
- **Acceptance criteria 2:** A site user can click on the comment thread to read the conversation.

### Account Registration

**As a Site User, I can register an account so that I can comment on a post.**

- **Acceptance criteria 1:** Given an email, a user can register an account.
- **Acceptance criteria 2:** Then the user can log in.
- **Acceptance criteria 3:** When the user is logged in, they can comment.

### Additional User Stories

**As a Site User, I can register for an account so that I can access additional features and interact with the community.**

- **Acceptance criteria 1:** A registration form is available with fields for username, email, and password.
- **Acceptance criteria 2:** Users receive a confirmation email upon successful registration.

**As a Site User, I can log in to my account so that I can access personalized features.**

- **Acceptance criteria 1:** A login form is available with fields for username and password.
- **Acceptance criteria 2:** Users are redirected to the home page upon successful login.

**As a Site User, I can log out of my account so that I can securely end my session.**

- **Acceptance criteria 1:** A logout button is available on all pages when the user is logged in.
- **Acceptance criteria 2:** Users are redirected to the home page upon successful logout.

**As a Site User, I can post comments on blog posts so that I can engage in discussions.**

- **Acceptance criteria 1:** A comment form is available on each post detail page.
- **Acceptance criteria 2:** Comments are displayed below the post in a threaded format.

**As an Admin, I can manage user comments so that I can moderate content on the site.**

- **Acceptance criteria 1:** Admins can view, approve, or delete comments from the admin dashboard.
- **Acceptance criteria 2:** Only approved comments are displayed to site users.

**As an Admin, I can create, edit, and delete blog posts so that I can manage site content.**

- **Acceptance criteria 1:** Admins have access to a content management interface.
- **Acceptance criteria 2:** Changes to posts are immediately reflected on the site.

**As a Site User, I can rate blog posts so that I can provide feedback on the content.**

- **Acceptance criteria 1:** A rating system (e.g., star ratings) is available on each post detail page.
- **Acceptance criteria 2:** Users can submit a rating along with an optional review comment.

**As a Site User, I can view the overall rating of a blog post so that I can gauge its quality.**

- **Acceptance criteria 1:** The average rating is displayed on each post detail page.
- **Acceptance criteria 2:** The number of ratings is also displayed.

**As a Site User, I can search for specific posts so that I can find content that interests me.**

- **Acceptance criteria 1:** A search bar is available on all pages.
- **Acceptance criteria 2:** Search results are displayed in a list format with post titles, authors, and brief summaries.

**As a Site User, I can view the profile of an author so that I can learn more about them.**

- **Acceptance criteria 1:** Author names on posts link to their profile pages.
- **Acceptance criteria 2:** Profile pages display the author’s bio, photo, and a list of their posts.


## Features

### Existing Features

- **Navigation Bar**
  - A fully responsive navigation bar with links to the Home, About, Contact, and user account pages. It is consistent across all pages for easy navigation.
 

- **Home Page**
  - Displays the latest blog posts with titles, authors, featured images, and excerpts.


- **About Page**
  - Provides information about FightVault’s mission, values, and goals.
    

- **Contact Page**
  - Allows users to get in touch with the FightVault team for inquiries or feedback.


- **User Authentication**
  - Includes user registration, login, and logout functionalities powered by Django Allauth.
  

- **Blog Post Detail Page**
  - Detailed view of each blog post, including the full content, author details, and the ability to leave comments and ratings.
  -

- **Comments and Ratings**
  - Users can leave comments and rate blog posts to provide feedback and engage with the content.
